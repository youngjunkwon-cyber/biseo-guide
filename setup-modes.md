# 설치 모드 비교 — 직접설치형 vs 승인형

← [메인 설치 가이드로 돌아가기](./)

설치 주체와 운영 방식에 따라 두 가지 모드가 있습니다.

- **A. 직접설치형** — 사용자 본인이 본인 PC에 설치 (가이드 그대로)
- **B. 승인형** — 용준님이 공용 PC에 설치, 사용자는 Slack 승인만

---

## 📢 공유용 (사용자/대표님한테 안내할 내용)

### A. 직접설치형 — 본인 PC에 직접 설치

사용자가 직접 체크·행동해야 할 것:

- [ ] 본인 PC (Win10+/macOS) + Chrome에 회사 Slack 로그인
- [ ] Claude Pro 계정 (본인 명의든 회사 지급이든)
- [ ] 가이드 따라 Claude Code 설치 → 붙여넣기 프롬프트 1회 실행
- [ ] 설치 중 브라우저에서 **Slack 앱 "허용" 1회 클릭**
- [ ] 가이드에 기재된 회사 Team API 키 복사·붙여넣기
- [ ] **본인 PC 항상 켜두기** + 절전/슬립 OFF
- [ ] (선택) 부팅 시 자동 실행 설정
- [ ] 에러·질문 시 용준님한테 Slack DM

### B. 승인형 — 공용 PC에 설치, 사용자는 승인만

사용자가 해야 할 것 (딱 2개):

- [ ] **본인 Slack user ID 알려주기** — 프로필 ⋯ → "멤버 ID 복사" (예: `U12AB3CD4`)
- [ ] 용준님이 보낸 **OAuth 링크 클릭 → "허용" 1회** (본인 Slack 로그인 상태에서)
- [ ] (선택) 긴급 처리할 주요 채널·VIP 인물 알려주기 — 안 주면 기본값으로 시작

**사용자 쪽 운영 책임: 없음.** PC 관리·재구동·업데이트 모두 용준님이 공용 PC에서 처리.

---

## 🛠 세팅용 (용준님이 할 일)

### A. 직접설치형 세팅 — 사용자가 본인 PC에 설치

**공통 선행**
- [ ] Anthropic 회사 Team 계정 API 키 발급
- [ ] biseo-guide README의 "API 키" 섹션에 실제 키 값 기재 + push
- [ ] 회사 Slack 워크스페이스에 비서 앱 생성·설치 (admin 승인 필요시 처리)
- [ ] `SLACK_BOT_TOKEN`, `SLACK_APP_TOKEN`, `SLACK_SIGNING_SECRET` 발급 확인

**사용자한테 공유**
- [ ] biseo-guide 링크 전달
- [ ] 설치 중 OAuth URL 전달 시점 조율 (본인 Slack 승인용)

**사후**
- [ ] 설치 지원·트러블슈팅 대응
- [ ] 사용자 PC 운영 상태 원격 점검 어려움 → 장애 시 대면·원격지원

### B. 승인형 세팅 — 공용 PC에서 구동

**공용 PC 준비**
- [ ] 항상 켜져 있는 PC 1대 확보
- [ ] 절전/슬립 모드 OFF, 디스플레이 끄기만 허용
- [ ] 부팅 시 자동 로그인 + `npm start` 자동 실행 설정 (Windows 작업 스케줄러 또는 `pm2`)
- [ ] 네트워크 안정적 (Slack + Anthropic API 도메인 접근 가능)

**공통 선행**
- [ ] Anthropic 회사 Team 계정 API 키 발급
- [ ] 회사 Slack 워크스페이스에 비서 앱 설치

**공용 PC에서 설치 실행**
- [ ] Claude Code 설치 + 본인 Claude Pro 계정으로 로그인
- [ ] biseo-guide 프롬프트 실행해서 설치 완료

**사용자한테 받기 & `.env` 기록**
- [ ] `OWNER_SLACK_USER_ID` — 사용자 user ID 받아서 `.env`에 기록
- [ ] `SLACK_USER_TOKEN` — 사용자한테 OAuth URL 전달 → 승인 받은 뒤 발급된 `xoxp-` 토큰 추출해서 `.env`에 기록
- [ ] `config.json`의 `channel_tiers`, `whitelist_user_ids` 세팅 (기본값으로 시작 가능)

**구동 확인**
- [ ] `⚡ 비서 앱 연결됨 (Socket Mode)` 로그 확인
- [ ] 사용자 Slack에 테스트 DM 보내서 비서봇 요약 카드 도착 확인
- [ ] 긴급/디제스트 알림 타이밍 샘플 확인

**사후**
- [ ] 공용 PC에서 Claude Code 로그아웃 (앱은 계속 구동)
- [ ] 주기적으로 앱 프로세스 살아있는지 모니터링 (`pm2 logs` 등)
- [ ] API 키 사용량·비용 모니터링 (Anthropic 콘솔)
- [ ] 이슈 발생 시 공용 PC에 접근해서 Claude Code 재로그인 → 디버깅

---

## 비교 요약

| 항목 | 직접설치형 | 승인형 |
|---|---|---|
| 사용자 부담 | PC 관리 필요 | 거의 없음 (승인 1회) |
| 용준님 운영 부담 | 낮음 (설치만) | 높음 (공용 PC 상시 관리) |
| 장애 대응 | 사용자 PC 원격 지원 | 용준님이 공용 PC에서 처리 |
| 사용자 PC 의존 | O (꺼지면 중단) | X (공용 PC가 대신 돌아감) |
| 추천 대상 | 개발 익숙한 사용자 | 비개발자 사용자 (대표님 등) |
